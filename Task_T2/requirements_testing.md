## Предложение по доработке.
-  ## Часть 1. UI
1. Предусловие: 
	 - Пользователь авторизован в сервисе Яндекс.Диск. 
	 - Сохранены два произвольных файла в аккаунте пользователя.
2. Реализована функция, осуществляющая поиск по именам файлов и папок(включая подпапки) в пределах текущего аккаунта Яндекс.Диск.
3. Поле поиска принимает любые символы Unicode(кириллица, латиница, цифры, спецсимволы). Максимальная длина принимаемой строки 255 символов. Спецсимволы не требуют экранирования.
4. Результат работы функции - список файлов и папок, содержащих в своем названии текст запроса.
5. Если файл не найден поиском по умолчанию, пользователь имеет возможность перейти в поиск по корзине нажатием на кнопку "Искать в корзине".
6. Вслучае, если нет файлов, удовлетворяющих запросу, получено сообщение "В вашем Диске не найдено файлов или папок по запросу <.......>"

-  ## Часть 2. API
1. Реализовано REST API PUT https://cloud-api.yandex.net:443/v1/disk/resources  с заголовками  
Content-Type: application/json, Authorization: [token] для создания папки в хранилище Яндекс.Диск.
2. Параметры строки запроса:  
    path - Путь к создаваемой папке,  
    fields - Список возвращаемых атрибутов (необязательное поле)
3. В случае успешного выполнения запроса получен ответ 201.
	  - Тело ответа следующего вида:  
	    {  
	    "href": "string",  
	    "method": "string",  
	    "templated": "boolean"  
	    }
      - В ключе ответа "href" вернулась ссылка, по которой  GET-запросом можно получить
   		созданную папку.
4. При возврате ответа от сервера отличному от 201 - папка создана не была, получено тело ответа 					   следующего вида:
{
 "message": "string",
 "description": "string",
 "error": "string"
 }
 Для кода 413 тело ответа возвращается тело ответа вида:
{
  "error": "string",
  "description": "string",
  "message": "string",
  "details": {},
  "reason": "string",
  "limit": 0
}
5. При неверном или отсутствующем токене получен ответ 401
  - Тело ответо следующего вида:
{
    "error": "UnauthorizedError",
    "description": "Unauthorized",
    "message": "Не авторизован."
}
6. При отсутствии обязательного параметра "path", получен ответ 400
  - Тело ответа следующего вида:
{
    "error": "FieldValidationError",
    "description": "Error validating field \"path\": This field is required.",
    "message": "Ошибка проверки поля \"path\": Это поле является обязательным."
}
7. При попытке создания папки по уже существующему пути получен ответ 409
  - Тело запроса следующего вида:
{
    "error": "DiskPathPointsToExistentDirectoryError",
    "description": "Specified path \"tmp_folder\" points to existent directory.",
    "message": "По указанному пути \"tmp_folder\" уже существует папка с таким именем."
}
