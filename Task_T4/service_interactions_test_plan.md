# План теcтирования интеграции сервисов A и B
## Объекты тестирования:
- Сервис A - принимает запросы REST, отдаёт amqp-команды Сервису B на подключение/отключение/блокировку услуги
- Сервис B - выполняет команды и сообщает в ответном amqp-сообщении результат операции
## Цели тестирования:
1. Убедиться, что Сервис A принимает запросы
2. Проверить упорядочивание Сервисом A amqp-команд, отправляемых на Сервис B
3. Убедиться, что сервис A принимает запросы, независимо от ответа Сервиса B
4. Убедиться, что статус услуги меняется только после получения положительного ответа от Сервиса B
5. Сервис A не отправляет следующую amqp-команду Сервису B, пока не получит ответ на предыдущую amqp-команду
## Описание тест-кейсов:
1. Проверить доступность Сервиса А для клиента
2. В Сервис A отправить подряд REST-запросы на:
    1. подключение услуги
    2. отключение услуги
    3. блокировку услуги
    - Первой в брокер сообщений уйдёт только команда 1.
    Только после получения от Сервиса B ответного сообщения, на него отправится команда 2.
    Команда 3 отправится только после получения ответа от Сервиса B на команду 2.
3. Сервис A отправляет в Сервис B команды:
    1. подключение услуги
    2. отключение услуги
    3. блокировка услуги
    - Сервис B выполняет команды и возвращает ответ об успешном выполнении.
    Статус услуги изменился.
    Сервис A отправляет клиенту ответ об успешном выполнении запроса.
4. В Сервис A отправить подряд REST-запросы на:
    1. подключение услуги
    2. отключение услуги
    3. блокировку услуги
    - Очередь amqp-команд Сервису B будет соответствовать очередности получения REST-запросов.
5. Сервис B недоступен/обрабатывает команду. В Сервис A отправить подряд REST-запросы на:
    1. подключение услуги
    2. отключение услуги
    3. блокировку услуги
    - Очередь amqp-команд Сервису B будет соответствовать очередности получения REST-запросов независимо от его доступности.
5. Сервис В упал во время обработки команды:
    - Статус услуги не изменился
    - После восстановления сервиса повтор выполнения команды или сообщение о неуспешном выполнении команды.
